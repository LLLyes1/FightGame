# M4 内容扩展与平衡迭代任务简报

## 1. 这份文档给谁

这份文档用于 M3 完成后的下一阶段交接。

M4 是项目第一次真正进入“内容生产阶段”。

它不是再补基础设施，也不是一口气塞很多内容，而是要用 M3 刚完成的能力：

- 内容接入模板
- 固定回归清单
- 参数集中区

来开展第一轮真正可控的内容扩展和版本平衡。

这份文档主要回答：

- M4 要先做什么
- 为什么按这个顺序做
- 每一波内容怎么加
- 每一轮验证怎么跑
- 平衡调整怎么收口

## 2. 当前阶段判断

默认前提：M3 已完成。

这意味着当前项目已经具备：

- 新内容标准接入路径
- 固定化回归流程
- 参数集中化调优入口

所以 M4 的核心问题不再是“基础够不够”，而是：

- 第一波新内容该怎么加最稳
- 每次扩内容后怎么快速验证旧内容没坏
- 怎么在小规模内容增长下建立第一轮真正的平衡方法

一句话：

**M4 是第一轮受控内容生产与平衡迭代阶段。**

---

## 3. M4 总目标

M4 要完成三件大事：

1. 做出第一波新增内容
2. 用固定回归制度保证版本稳定
3. 用参数中心完成第一轮正式平衡迭代

建议把 M4 的阶段成果定义为：

- 新增 `1 把武器`
- 新增 `1 个角色`
- 所有合法组合都通过回归
- 第一轮平衡调整完成

也就是说，M4 不追求“量大”，而追求：

- 扩一次内容真的跑通
- 调一次平衡真的可控
- 版本开始具备持续迭代节奏

---

## 4. 当前阶段先不要做什么

M4 虽然进入内容生产，但依然不建议：

- 一次性新增很多角色
- 一次性新增很多武器
- 同时改很多基础系统
- 一边扩内容一边重做 UI / 地图 / 核心手感

原因很简单：

- 现在最需要建立的是“可持续版本循环”
- 不是“做出一堆还没被验证过的内容”

原则：

**一次只扩大一小步，但每一步都稳定落地。**

---

## 5. M4 推荐阶段结构

建议把 M4 分成 4 个工作块：

1. 第一波武器扩展
2. 第一波角色扩展
3. 组合回归与问题修复
4. 第一轮平衡迭代收口

对应建议任务编号：

- `T11` 第一波新武器接入
- `T12` 第一波新角色接入
- `T13` 内容扩展后的组合回归
- `T14` 第一轮平衡调优

## 6. T11 第一波新武器接入

### 目标

先新增一把武器，验证 M3 做好的接入模板、回归流程和调优流程是否真的可用于生产。

### 为什么先做武器

建议优先做武器而不是角色，原因是：

- 武器通常比角色更容易控制变量
- 可以先复用现有两个角色来验证武器差异
- 更容易快速看出新增内容是否真正改变战斗节奏

### 建议新增类型

不要做和现有两把武器太像的东西。

当前已有：

- `saber`：近战压制
- `hand_cannon`：远程牵制

所以第一把新武器建议从下面选一个方向：

1. 中距离控制型武器
   例如：长枪 / 链刃
2. 慢启动高终结型武器
   例如：重锤
3. 高风险爆发型武器
   例如：近爆枪 / 充能炮

推荐优先：

- **长枪或重锤**

原因：

- 差异更清楚
- 更容易观察到攻击距离、节奏、击飞风格变化

### 建议拆卡

1. `T11-C1` 选定武器定位与战斗语言
2. `T11-C2` 建立完整武器数据条目
3. `T11-C3` 让两名现有角色都能使用它
4. `T11-C4` 跑武器专项回归
5. `T11-C5` 做第一轮武器节奏调整

### 具体要做什么

#### T11-C1 选定武器定位

必须先回答：

- 这把武器靠什么建立优势
- 它的节奏是快、慢、压制、控制还是终结
- 它最强的 2 个场景是什么
- 它最怕的 2 个场景是什么

建议产物：

- 一段武器定位说明
- 关键攻击槽位差异说明

#### T11-C2 建完整武器条目

使用 [content-integration-template.md](/E:/MyGame/Fight/docs/content-integration-template.md)。

要求：

- 所有攻击槽位齐全
- 命名风格统一
- 先保证“都有动作”，再追求“都很复杂”

优先关注文件：

- [weapon_database.gd](/E:/MyGame/Fight/scripts/core/weapon_database.gd)

#### T11-C3 双角色接入验证

至少用：

- `vanguard + 新武器`
- `sky_drifter + 新武器`

去看两件事：

- 角色差异是否仍然成立
- 武器差异是否确实被读出来

#### T11-C4 武器专项回归

基于 [regression-test-checklist.md](/E:/MyGame/Fight/docs/regression-test-checklist.md) 增加专项验证：

- 全部攻击槽位可触发
- 命中反馈正常
- 近战/投射物逻辑正常
- 不会软锁状态

#### T11-C5 第一轮武器节奏调整

使用 [tuning-parameter-hub.md](/E:/MyGame/Fight/docs/tuning-parameter-hub.md) 指导改动。

优先调整：

- `startup`
- `active`
- `recovery`
- `damage`
- `base_knockback`
- `knockback_growth`

### 验收标准

- 新武器完整接入成功
- 两名旧角色都能稳定使用
- 武器风格与现有两把武器明显不同
- 跑完专项回归后无阻塞问题

## 7. T12 第一波新角色接入

### 目标

新增一个角色，验证“角色身份扩展”是否已经足够稳定。

### 为什么第二步才做角色

因为角色变量更复杂，会影响：

- 移动
- 跳跃
- 冲刺
- 回场
- 受击观感

先做武器能把风险分层，等武器扩展验证通过，再加角色更稳。

### 建议新增类型

当前已有：

- `vanguard`：标准型
- `sky_drifter`：高机动 / 滑翔型

所以新角色建议优先补一个差异明显的空位：

1. 重型角色
   特点：更重、更稳、击飞强、回场弱
2. 控场角色
   特点：节奏慢，但范围和站位强
3. 高爆发脆皮角色
   特点：动作快，但容错低

推荐优先：

- **重型角色**

原因：

- 和现有角色差异最大
- 最容易建立“移动身份”辨识
- 也最适合检验击飞、压边、回场平衡

### 建议拆卡

1. `T12-C1` 选定角色身份
2. `T12-C2` 建立角色数据条目
3. `T12-C3` 用两把现有武器分别验证
4. `T12-C4` 跑角色专项回归
5. `T12-C5` 做第一轮角色手感调整

### 具体要做什么

#### T12-C1 角色身份定义

必须先回答：

- 这个角色如何移动
- 这个角色如何回场
- 这个角色如何赢
- 这个角色如何输

建议产物：

- 一段角色定位说明
- 一段移动身份说明

#### T12-C2 建角色数据条目

使用 [content-integration-template.md](/E:/MyGame/Fight/docs/content-integration-template.md)。

优先关注文件：

- [character_database.gd](/E:/MyGame/Fight/scripts/core/character_database.gd)
- [dash_config.gd](/E:/MyGame/Fight/scripts/core/dash_config.gd)

重点字段：

- `move_speed`
- `air_speed`
- `gravity`
- `jump_count`
- `jump_velocity`
- `weight`
- `knockback_resistance`
- `dash_config_overrides`

#### T12-C3 双武器验证

至少验证：

- `新角色 + saber`
- `新角色 + hand_cannon`

如果 `T11` 已经完成，也建议补测：

- `新角色 + 新武器`

#### T12-C4 角色专项回归

重点看：

- 出生与回场是否稳定
- 重量变化是否影响受击逻辑
- 冲刺覆盖是否正常
- HUD 文本与组合信息是否正确

#### T12-C5 第一轮角色手感调整

优先从以下维度调：

- 地面移动是否过肉
- 空中控制是否过飘
- 回场是否过强或过弱
- 重量是否让对局失衡

### 验收标准

- 新角色完整接入成功
- 至少能稳定使用两把现有武器
- 和现有两名角色移动身份明显不同
- 回归通过后无严重软锁或节奏崩坏

## 8. T13 内容扩展后的组合回归

### 目标

把新增内容纳入正式版本矩阵，确认项目从“原始 4 组合”升级到“扩展后组合”时仍可控。

### 建议思路

不要一口气把所有新旧组合全部深测到同等强度，而是分两层：

1. 固定基础矩阵回归
2. 新增内容专项矩阵回归

### 建议拆卡

1. `T13-C1` 跑基础四组合固定回归
2. `T13-C2` 跑新武器专项矩阵
3. `T13-C3` 跑新角色专项矩阵
4. `T13-C4` 汇总问题并分级

### 具体要做什么

#### T13-C1 基础四组合回归

先继续跑旧四组合：

- `vanguard + saber`
- `vanguard + hand_cannon`
- `sky_drifter + saber`
- `sky_drifter + hand_cannon`

目的：

- 验证新增内容没有破坏旧内容

#### T13-C2 新武器专项矩阵

新增武器至少要测：

- `vanguard + 新武器`
- `sky_drifter + 新武器`

如有新增角色，再加：

- `新角色 + 新武器`

#### T13-C3 新角色专项矩阵

新增角色至少要测：

- `新角色 + saber`
- `新角色 + hand_cannon`

#### T13-C4 问题分级

建议按三类记：

- 阻塞问题：会影响继续测试或发布
- 平衡问题：能玩，但明显失衡
- 体验问题：不影响功能，但手感或可读性不理想

建议产物：

- 一轮正式回归记录
- 一个问题清单

### 验收标准

- 旧四组合未被新内容破坏
- 新内容进入矩阵后没有阻塞级问题
- 已有问题被明确记录并分级

## 9. T14 第一轮平衡调优

### 目标

基于 M4 第一波新增内容，完成项目的第一轮真正意义上的“平衡迭代”。

这一步不是追求完全平衡，而是建立一个稳定方法：

- 看什么
- 先改什么
- 一次改多少
- 改完怎么确认

### 建议拆卡

1. `T14-C1` 明确平衡观察维度
2. `T14-C2` 建立本轮优先级
3. `T14-C3` 做小步参数迭代
4. `T14-C4` 做调优后确认回归

### 具体要做什么

#### T14-C1 明确平衡观察维度

至少看：

- 角色移动身份是否清晰
- 武器攻击节奏是否清晰
- 是否存在一套明显无脑优势组合
- 是否存在完全不值得选的组合
- 回场能力是否差异过大
- 边缘压制是否过度失衡

#### T14-C2 建立本轮优先级

建议调整顺序：

1. 先修阻塞体验问题
2. 再修明显失衡问题
3. 最后修细节手感问题

不要反过来：

- 不要先抠细节，再放着大失衡不管

#### T14-C3 小步参数迭代

严格使用 [tuning-parameter-hub.md](/E:/MyGame/Fight/docs/tuning-parameter-hub.md)。

建议原则：

- 一次只改一层
- 一次只改少量关键参数
- 每次改完就验证

优先可调内容：

- 角色：速度、重力、重量、跳跃、冲刺覆盖
- 武器：启动、收招、击飞、命中收益
- 全局：反馈时长、镜头、消息时间、缓冲窗口

#### T14-C4 调优后确认回归

每轮调参之后都要跑：

- 至少一轮基础四组合
- 至少一轮新增内容专项矩阵

目的：

- 防止调一个组合，带坏另一个组合

### 验收标准

- 没有明显“必选组合”或“废弃组合”
- 角色和武器差异都更清楚
- 调整过程有记录、有依据、有回归

## 10. 推荐顺序

请按这个顺序推进 M4：

1. 先做 `T11` 新武器
2. 再做 `T12` 新角色
3. 然后做 `T13` 组合回归
4. 最后做 `T14` 第一轮平衡调优

推荐原因：

- 武器先扩更便于控风险
- 角色后扩更能看出基础系统是否真的稳
- 回归要在内容接入后集中跑一轮
- 平衡应建立在内容完整接入之后

最短理解版：

**先加武器，再加角色，再验证矩阵，最后调第一轮平衡。**

## 13. 阶段验收口径

只有同时满足下面 3 类条件，才建议认为 M4 完成。

### 13.1 内容扩展通过

- 至少新增 1 把武器
- 至少新增 1 个角色
- 新内容都通过标准接入流程

### 13.2 版本稳定通过

- 基础旧四组合回归通过
- 新内容专项矩阵回归通过
- 没有阻塞问题残留

### 13.3 平衡迭代通过

- 完成一轮正式参数调优
- 角色和武器差异更清楚
- 没有特别明显的坏组合、废组合或无脑强组合

## 15. 给持续推进的最后一句话

M4 是你们第一次真正进入版本迭代期。

不要把它做成“大批量堆内容”，而要把它做成：

**小步扩内容 -> 固定回归 -> 小步调平衡 -> 再扩下一步**

只要这条节奏建立起来，后面项目就会从“原型开发”转进真正的“版本开发”。
