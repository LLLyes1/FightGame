# M5 人工试玩脚本

## 1. 目的

这份脚本用于把当前 `3 角色 x 3 武器` 版本的手测流程固定下来，避免每次试玩都临场想测什么。

本轮重点不是扩内容，而是确认：

- 旧四组合没有被新内容带偏
- `halberd` 的压边与反空没有超出健康区间
- `bastion` 的重型身份成立，但回场仍然可用
- 当前中文菜单、HUD、训练读数在真实试玩中可读

## 2. 手测前准备

- 先运行一次自动回归：`res://scripts/tools/m5_regression_runner.gd`
- 先看主菜单预览是否正常显示：
  - 角色名
  - 武器名
  - 距离/节奏标签
  - 角色一句话定位
  - 武器一句话定位
- 进入训练模式前确认 HUD 能看到：
  - 当前组合
  - 当前状态
  - 当前招式
  - 命中/受击记录
  - 跳跃和闪避资源

## 3. 固定试玩顺序

### 3.1 基础四组合

按下面顺序各玩 1 轮训练 + 1 轮对战观察：

1. `vanguard + saber`
2. `vanguard + hand_cannon`
3. `sky_drifter + saber`
4. `sky_drifter + hand_cannon`

每轮重点看：

- 进局流程是否顺手
- 菜单预览与实际手感是否一致
- 地面轻攻击 / 地面重攻击 / 上下地面技是否稳定触发
- 空中招式与方向必杀是否和 HUD 提示一致
- 回场、出界、结算、返回菜单是否清楚

### 3.2 新内容专项五组合

按下面顺序各玩 1 轮训练 + 1 轮对战观察：

1. `vanguard + halberd`
2. `sky_drifter + halberd`
3. `bastion + saber`
4. `bastion + hand_cannon`
5. `bastion + halberd`

每轮重点看：

- `halberd` 是否能靠单一长判定无脑压边
- `halberd` 的重攻击、侧必杀、上地面技、上空技是否风险足够
- `bastion` 是否能靠基本跳跃和空中闪避回到台上
- `bastion` 在远程压力下是否仍有最小可用回场空间
- 镜头、HUD、战斗播报在新旧组合碰撞时是否仍稳定

## 4. 每轮固定动作

每个组合都至少做下面动作：

1. 训练模式下把所有地面招式打一遍。
2. 训练模式下把所有空中招式打一遍。
3. 训练模式下分别测试：
   - 上地面技
   - 下地面技
   - 中立必杀
   - 侧必杀
   - 上必杀
   - 下必杀
4. 人为制造一次回场压力，确认资源与 HUD 读数是否一致。
5. 进入对战，重点观察 2-3 次命中、一次压边、一次出界和一次重开流程。

## 5. 记录模板

每个组合都按下面模板记录：

| 字段 | 记录内容 |
| --- | --- |
| 测试组合 | 例如 `bastion + halberd` |
| 测试模式 | `training` / `versus` |
| 观察目标 | 例如“回场是否过弱” |
| 现象 | 看到什么 |
| 是否稳定复现 | `是 / 否 / 偶发` |
| 问题级别 | `阻塞 / 平衡 / 体验` |
| 建议入口 | `character_database.gd / weapon_database.gd / hud.gd / main.gd / game_scene.gd` |

## 6. 优先级判断口径

- `阻塞`：会让试玩无法继续，或会导致错误理解当前组合。
- `平衡`：能玩，但明显过强、过弱或过于无脑。
- `体验`：功能正确，但菜单、HUD、播报、镜头或中文排版不舒服。

## 7. 本轮重点观察项

- `halberd`
  - `heavy_ground`
  - `special_side`
  - `up_ground`
  - `up_air`
- `bastion`
  - 空中移动修正
  - 双跳后回场窗口
  - 空中闪避后回场余量
- 中文界面
  - 主菜单预览是否拥挤
  - 训练 HUD 是否信息够用但不过载
  - 暂停与结算消息是否有截断或遮挡
