## 3. 当前最终风险清单

### 高优先级

- 缺少九组合的正式人工试玩结论。
- `bastion + halberd` 是否会在真实对打里形成过稳的压线压边节奏，尚未人工确认。

### 中优先级

- `sky_drifter + hand_cannon` 的拉扯安全性是否偏高，尚未人工确认。
- `bastion + hand_cannon` 的回场代价是否仍偏高，尚未人工确认。

### 低优先级

- 主菜单预览和训练 HUD 的长文本是否仍有局部拥挤，需要一次真实 UI 复核。

## 4. 下一轮扩展前必须完成的事

1. 按 [m5-manual-playtest-script.md](/E:/MyGame/Fight/docs/m5-manual-playtest-script.md) 跑完九组合人工试玩。
2. 把结果填入 [m5-manual-playtest-report.md](/E:/MyGame/Fight/docs/m5-manual-playtest-report.md)。
3. 如果真人试玩确认高优先级风险已下降到可接受，再重新做一次 gate review。

## 5. 为什么这次不直接判“可以扩展”

因为当前最缺的不是代码功能，而是**真实长对局证据**。

继续在没有正式人工试玩的情况下扩内容，会把下一阶段建立在不完整基线上，届时很难区分：

- 是新内容本身有问题
- 还是旧阵容本来就没收口

所以本轮最合理的 PM 结论不是“马上扩”，而是：

**先用这版更清晰的菜单/HUD 与更稳一点的新内容参数，跑完一轮正式手测，再决定 M6。**
