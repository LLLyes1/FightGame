# M6 UI、视觉与资产接入程序简报

## 1. 这份文档给谁

这份文档给负责 M6 程序实现的人。

目标不是重复讲“为什么要升级 UI 和视觉”，而是明确：

- 当前工程哪些地方还是原型结构
- 程序侧要补哪些接入口
- 怎样让 UI 和真实资源资产可以逐步接进来

---

## 2. 当前程序结构判断

从当前工程看，UI 和视觉表现仍然明显带有原型特征：

### 菜单层

- [main.tscn](/E:/MyGame/Fight/scenes/main/main.tscn) 是基础容器结构
- [main.gd](/E:/MyGame/Fight/scripts/main/main.gd) 里动态创建了角色、武器选择器和文本预览

### 局内 HUD

- [hud.gd](/E:/MyGame/Fight/scripts/game/hud.gd) 通过代码直接创建面板和标签

### 角色表现

- [fighter.gd](/E:/MyGame/Fight/scripts/actors/fighter.gd) 主要通过 `_draw()` 和 `draw_rect()` 表现角色

### 投射物表现

- [projectile.gd](/E:/MyGame/Fight/scripts/actors/projectile.gd) 主要通过程序绘制

### 地图表现

- [stage.gd](/E:/MyGame/Fight/scripts/game/stage.gd) 主要通过程序绘制背景、平台和装饰

### 当前缺口

- 没有正式 `assets` 目录
- 没有统一的资源加载入口
- 数据库没有视觉资源字段
- 局内没有武器视觉挂点
- 菜单没有卡片式资源位

---

## 3. M6 程序侧要解决的三件大事

## 3.1 把 UI 从文本原型升级成结构化界面

程序目标：

- 从“动态下拉框 + 文本说明”升级到“卡片选择 + 摘要面板 + 结果展示”

## 3.2 把视觉资产接入链搭起来

程序目标：

- 允许角色、武器、地图、UI 资源通过路径和配置接入

## 3.3 给局内表现留出资源化入口

程序目标：

- 允许战斗表现逐步从 `draw_rect()` 迁移到贴图、图标、特效、分层资源

---

## 4. T21 主菜单与选择页 UI 改版

## T21-C1 固定菜单场景结构

### 目标

减少 `main.gd` 里“运行时全动态搭建 UI”的比重，把可视结构尽量回到场景里。

### 主要文件

- [main.tscn](/E:/MyGame/Fight/scenes/main/main.tscn)
- [main.gd](/E:/MyGame/Fight/scripts/main/main.gd)

### 程序要做的事

- 在场景里预留角色区、武器区、地图区、摘要区、操作区
- 把当前主要靠 `Label` 拼接的结构变成稳定节点树

### 完成标准

- 主菜单结构主要在场景中可见
- `main.gd` 负责数据绑定，而不是负责大部分布局生成

---

## T21-C2 角色与武器卡片化

### 目标

让角色、武器不再只通过 `OptionButton` 被选择。

### 程序要做的事

- 为角色卡片预留：
  - 名称
  - 定位短句
  - 头像/占位图
  - 选中态
- 为武器卡片预留：
  - 名称
  - 类型标签
  - 图标/占位图
  - 选中态

### 完成标准

- 玩家可以直观看到当前所选内容
- 没有真实图片时也能显示统一占位样式

---

## T21-C3 地图选择与预览位

### 目标

为地图正式扩张留出 UI 落点。

### 程序要做的事

- 支持地图列表
- 支持地图名称、说明、预览缩略位
- 支持地图选中态

### 完成标准

- 菜单不再默认只有一张图的隐含逻辑

---

## T21-C4 当前组合摘要面板

### 目标

让玩家一眼看懂当前 loadout 的基本风格。

### 程序要做的事

- 支持显示角色定位
- 支持显示武器定位
- 支持显示组合标签
- 支持显示推荐难度或推荐人群

### 完成标准

- 当前组合信息不再依赖大段说明文字拼接

---

### 程序要做的事

## T22-C3 局内提示与战斗反馈升级

### 目标

## 6. T23 视觉资产接入底层

## T23-C1 建立正式资源目录

### 目标

让资源接入不再零散。

### 建议目录

- `assets/ui/`
- `assets/characters/`
- `assets/weapons/`
- `assets/stages/`
- `assets/fx/`

### 完成标准

- 工程内有正式资源目录结构

---

## T23-C2 扩展角色与武器数据库字段

### 目标

让内容数据不仅描述玩法，也描述视觉入口。

### 主要文件

- [character_database.gd](/E:/MyGame/Fight/scripts/core/character_database.gd)
- [weapon_database.gd](/E:/MyGame/Fight/scripts/core/weapon_database.gd)

### 建议新增字段

角色：

- `portrait_path`
- `card_art_path`
- `body_sprite_path`
- `accent_fx_color`

武器：

- `icon_path`
- `card_art_path`
- `battle_sprite_path`
- `trail_fx_color`

### 完成标准

- 数据库支持描述资源路径和表现字段

---

## T23-C3 资源加载与缺省回退机制

### 目标

保证“有资产时显示资产，没有资产时也不会崩”。

### 程序要做的事

- 封装统一的加载函数
- 路径不存在时回退到默认占位图
- 占位图缺失时回退到纯色样式

### 完成标准

- UI 和战斗表现都能容忍资源不完整状态

---

## 7. T24 武器与角色视觉接入

### 目标

### 程序要做的事

- 武器卡支持图标或卡图

### 完成标准

- 菜单视觉表达不再主要靠文字

---

## T24-C2 局内武器视觉接入

### 目标

让武器在战斗中也有真实存在感。

### 主要文件

- [fighter.gd](/E:/MyGame/Fight/scripts/actors/fighter.gd)

### 程序要做的事

- 增加武器显示节点或挂点
- 按武器类型加载不同资源

### 完成标准

## T24-C3 角色视觉第一版接入

### 目标

让角色不再只是一块纯绘制体。

### 程序要做的事

- 为角色增加显示层节点
- 支持角色身体贴图或头像化局内表现
- 保留当前 `draw_rect()` 作为应急回退

### 完成标准

- 角色视觉具备从程序体到资源体的过渡能力
